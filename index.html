HTML

  • The head element contains information about the webpage
  • The body element represents the viable content shown to the user
  • The section element organizes the content on the page based on the topic
  • Connect the HTML and CSS file with "link rel="stylesheet" href="./assets/style.css" in the head section

CSS

  • Selector: element or attribute being modified
  • Property: the CSS property to apply to the element
  • decloration: the property and its value seperated by a colon
  • Margin: outside of the element
  • Padding: inside of the element
  • * is the special wild card symbol used to apply rule(s) to all the elements
  • Use .class-name to designate a class selector

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • class .: opens Visual Studio Code
  • git add -A: adds changed files in the working directory to the staging area of git
  • git commit -m "commit message": commit changes to the repo
  • Commit messages should be short descriptive messages of the changes implemented
  • git pull origin main: ensure that my current branch is up to date
  • git push origin feature/branch-name: sends changes to github
  • Pull requests: base branch should be the one to merge into and compare branch should be the feature branch. Remember to give a desciption of changes
  • touch file-name: create new file in the current folder
